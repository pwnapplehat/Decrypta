Multiple Apple IDs, contained/cleanable cache, sign-in indicator

- Multiple Apple ID accounts: each Apple ID gets its own isolated ipadecrypt
  root (accounts\<slug> with its own config + cookies). Add / switch / sign out /
  remove from the Sign in tab; legacy single-account state is migrated on upgrade.
- Clear signed-in indicator in the header (green dot + active Apple ID, or grey
  "not signed in") plus per-account state in the account switcher.
- Encrypted-download cache now lives INSIDE the chosen output folder
  (<output>\.decrypta-cache via a no-admin junction) instead of LocalAppData/temp,
  so everything is contained. Credentials stay in LocalAppData.
- One-click "Clean cache" (Settings + Library) wipes cached and partial (.tmp)
  downloads completely; a failed/cancelled decrypt auto-clears its partial. Cache
  size is shown in both places.
- Choose the output folder from the Library tab too (not just Settings).
- Core: AccountService + CacheManager, engine shares the app Settings; tests +15.
